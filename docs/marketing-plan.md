# 拡散・バズ化計画

LDK Lighting Lab を無料公開後に広めるための実行計画。README の公開方針（無料・ログインなし・広告なし・透かし入り書き出し）と、画面の免責表示（照度・配光を保証しない）を前提とする。

## 1. コアメッセージ

> **「新築の後悔ランキング常連の『照明』を、契約前に自分の間取りで夜シミュレーションできる無料ブラウザツール」**

- 「照度計算ツール」ではなく「夜の見え方を事前に比べる道具」として一貫させる（share-demo.md の方針を踏襲）。
- フックは「後悔の予防」。家づくり界隈では「入居後に照明で後悔した」投稿が定番コンテンツであり、その解決策として刺さる。
- すべての投稿に免責の一言を添える：「実照度を保証するものではなく、雰囲気比較用です」。誠実さ自体が界隈での信頼＝拡散力になる。

## 2. ターゲットと拡散の2軸

| 軸 | 対象 | 刺さる切り口 | 主戦場 |
|---|---|---|---|
| **施主軸（主）** | 注文住宅・リノベ検討中の施主 | 「自分の間取り図（PDF可）を読み込んで夜を見られる」 | X の家づくり界隈、Instagram `#家づくり` `#注文住宅` `#照明計画`、ルームクリップ |
| **技術軸（副）** | Web/3D エンジニア | 「ブラウザだけでパストレーシング（three-gpu-pathtracer）、WYSIWYG編集」 | Zenn / はてなブックマーク / X の技術界隈 / Hacker News (Show HN) |

技術軸は直接のユーザーにはならないが、はてブ・HN 経由の流入とドメイン信頼を作り、施主軸の投稿が「怪しい業者ツール」扱いされるのを防ぐ。

## 3. バイラルループ（プロダクトに内蔵済みの仕組みを使う）

1. ユーザーが自分の間取りで夜の比較画像を作る
2. PNG 書き出し・比較一覧の画像には **アプリ名＋URL の透かし** が入る
3. 家づくり界隈は Before/After・比較画像文化が強く、SNS に貼られる
4. 透かしから新規ユーザーが流入 → 1 に戻る

つまり **「シェアしたくなる比較画像を最短で作れること」が拡散の本体**。投稿施策はこのループの着火剤と位置づける。

## 4. 公開前の必須準備（プロダクト側）

| 項目 | 内容 | 状態 |
|---|---|---|
| 公開URL固定 | `VITE_APP_URL` をビルドに設定し透かしを固定URL化 | README記載・要設定 |
| OGP | `index.html` に og:title / og:image（夜のLDKレンダー画像）/ description を設定。X カードで画像が出ないと拡散効率が大きく落ちる | 要確認 |
| ワンタップデモ | `share-demo-project.json` をURLパラメータ等で自動読込できる導線（「まず触れる」までの摩擦を最小化） | 要検討 |
| モバイル動作 | 界隈の閲覧はスマホ中心。2D/3D 操作の直近修正を実機確認 | 対応中 |
| フィードバック導線 | 💬フォーム→GitHub Issue は稼働確認済みにしておく | 実装済み |

## 5. コンテンツの型（勝ちパターン候補）

1. **Before/After 動画（15〜30秒）**: 同じLDKで「昼白色ダウンライトだけ」→「電球色＋間接照明」を切替。テキストは「この差、入居後に気づくと遅い」。X / Reels / TikTok / Shorts に同素材を展開。
2. **「実況」スレッド**: 架空デモ間取りのPDF読込→壁なぞり→照明配置→夜レンダーまでをスクショ連投。「無料・ログイン不要・ブラウザだけ」を最後に明記。
3. **失敗共感型**: 「ダウンライト等間隔グリッド配置」vs「タスク位置狙い配置」の比較画像。界隈で議論が起きやすいテーマ＝リプライが伸びる。
4. **技術記事（Zenn）**: 「ブラウザでパストレーシングして照明シミュレーターを作った」— WYSIWYG常駐パストレ、three-mesh-bvh worker、色温度→RGBマッピングの話。末尾にアプリへのリンク。
5. **色温度クイズ**: 「2700K/3500K/5000K、どれが好み?」の3枚比較画像で投票を取る。参加コストが低くエンゲージメントが稼げる定番形式。

素材はすべて `docs/share-demo.md` の架空デモ間取りを使う（実在物件の図面は使わない）。

## 6. ローンチ手順（時系列）

**Phase 0（公開前 1週間）**
- 上記4の準備を完了。デモ動画・比較画像を各3〜5点ストック。
- 投稿用アカウント（X）を整備。プロフィールに「個人開発・無料・雰囲気比較用」と明記。

**Phase 1（ローンチ週）**
- Day 1: X にローンチ投稿（30秒デモ動画＋「無料・ログイン不要」＋URL）。同日に Zenn 技術記事を公開し技術界隈からの初速を作る。
- Day 2-3: Instagram Reels / TikTok に同素材を展開。
- Day 4-7: 型3（比較議論ネタ）と型5（クイズ）で会話を発生させる。エゴサして自分の間取りで試した投稿には必ず反応する。

**Phase 2（2〜4週目）**
- 家づくり系の発信者（ブロガー・インスタグラマー・一級建築士系YouTuber）数名に DM で紹介。「無料ツールなので紹介料なし、感想だけ欲しい」スタンス。金銭が絡まない方が界隈では信頼される。
- Show HN 投稿（"I built a browser-based home lighting simulator with real-time path tracing"）。
- フィードバック Issue から要望上位を1つ実装して「ユーザーの声で進化してる」更新投稿。

**Phase 3（継続運用）**
- 週1〜2本、型1/3/5 のローテーション投稿。
- ユーザー投稿（透かし付き画像）のリポスト・引用を最優先。UGC が回り始めたら投稿頻度より反応速度に軸足を移す。

## 7. 計測と判断基準

- **一次指標**: 公開URLへのユニーク訪問（Cloudflare Analytics）、デモJSON読込数。
- **バイラル指標**: 透かし付き画像を含む第三者投稿の件数（エゴサ: アプリ名＋URL）。
- **質の指標**: 💬フォーム経由の Issue 数と内容。
- 判断: Phase 1 で第三者投稿がゼロなら、施策より「共有画像を作るまでの摩擦」（4のワンタップデモ、書き出し導線）を先に改善する。README の収益化判断（使われ方を見てから）もこの計測に乗せる。

## 8. やらないこと

- 実照度・施工品質を示唆する表現（「これで失敗しない」等の断定コピー）— 免責と矛盾し炎上リスク。
- 実在住宅・不動産図面をデモ素材に使うこと。
- 広告出稿・インフルエンサーへの金銭依頼（初期は無料ツールの善意ドリブンで通す）。
- ログイン必須化などバイラルループの摩擦を増やす変更。
