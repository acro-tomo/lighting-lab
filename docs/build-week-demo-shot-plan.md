# Build Week demo shot plan

This plan is timed to `docs/build-week-demo-narration-en.txt` and the generated
Kokoro narration. It is a capture plan, not a product claim.

| Narration timing | Shot and action | What the viewer should notice |
| --- | --- | --- |
| 0:03–0:37 | Wide two-dimensional plan and three-dimensional room; briefly reveal the floor-plan import action. | The app starts from a homeowner's floor plan rather than a specialist CAD workflow. |
| 0:37–0:51 | Select **Dining pendant, west** in the two-dimensional plan, then hold a close three-dimensional view of the dining table. Change **2700K → 3500K** in the visible inspector. | The same dining-table fixture is selected, and the colour change is judged where it matters. |
| 0:51–1:14 | Keep the same dining-table close view. Adjust brightness, drag the pendant in the plan, then adjust its spread. | Brightness, position, and distribution are alternatives a homeowner can compare quickly. |
| 1:14–1:19 | Move to a second three-dimensional viewpoint around the dining area. | A plan edit can be judged from more than one viewpoint. |
| 1:19–1:23 | Hold the completed **High** quality Finished Look render from this dining viewpoint. | Finished Look is shown only after rendering has completed; no progress frame or black frame is used. |
| 1:32–1:41 | Select the living-room sculpture spotlight and change its aim direction with the aim dial. | The app supports more than a single pendant adjustment. |
| 1:42–1:51 | Show a project save state and a phone-sized 2D/3D control view. | The workflow is available on desktop and phone. |
| 1:55–2:24 | Return to wide plan/3D view, ending on the completed dining Finished Look image. | The closing claim reconnects the choices to the homeowner's own plan. |

The video does not call the output physically exact and does not describe it as
a guaranteed illuminance result. There is no background music; the narration
and the visible colour comparison need to remain clear.
